# Sleep Health & Night Memo Skill

## 核心规则

### 1. 夜间提醒（触发条件）

收到用户消息时，检测当前**当地时间**（自动获取用户时区）：

- 如果是 **22:00 - 06:00**，在回复前添加健康提醒
- 提醒语言跟随**用户发送消息的语言**（用户用中文就中文提醒，用户用英文就英文）

### 2. 夜间备忘录（核心功能）

如果用户在夜间表达了：
- 一个想法、点子、计划
- 一个想做的任务
- 一个待处理的事项

**且没有明确说"现在就做"/"立刻"/"urgent"等**，执行以下操作：

```
在 /data/openclaw/.openclaw/workspace/scripts/night_memos/YYYY-MM-DD.md 中记录：
- 时间（当地时区）
- 用户表达的内容摘要
- 原文
```

然后在回复中加入：
> "💡 您的想法不错，我先记到备忘录里，明早 09:00 给您提醒。"

**只有用户明确说"现在就做"才立即执行。**

### 3. 早晨提醒（HEARTBEAT 触发）

每天 **09:00-09:30**（当地时间）检查是否有昨夜（22:00后）的备忘录，如有则提醒用户：

```
🌙 昨夜备忘录提醒：
[列出所有未处理的夜间想法]
```

提醒使用**用户偏好的语言**（根据用户历史消息判断）。

## 实现细节

### 时区获取

使用消息时间戳中的时区信息（如有）；若无，根据用户历史判断（默认尝试自动检测）。

### 备忘录文件格式

```markdown
# 夜间备忘录 — YYYY-MM-DD

## 01:23
**想法摘要**
用户原文：...
```

### 判断是否为"想法"

关键词信号（出现则视为想法）：
- "我想"/"我想做"/"我觉得"/"要不要"
- "should"/"maybe"/"could"/"might"/"idea"
- "later"/"tomorrow"/"eventually"
- "备忘"/"记一下"/"提醒我"

明确"立刻做"信号（不记备忘录，立即执行）：
- "现在"/"立刻"/"马上"/"马上做"/"现在就"
- "urgent"/"asap"/"right now"/"immediately"

### 备忘录位置

`/data/openclaw/.openclaw/workspace/scripts/night_memos/YYYY-MM-DD.md`

## 触发条件速查

| 场景 | 行为 |
|------|------|
| 22:00-06:00，表达了想法 | 记备忘录 + 回复加提醒 |
| 22:00-06:00，明确说"现在就做" | 立即执行，不记备忘录 |
| 06:00-22:00 | 正常处理，无特殊行为 |
| 09:00 检查有昨夜记录 | 发送早晨提醒 |
